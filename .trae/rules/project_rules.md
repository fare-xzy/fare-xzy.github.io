1. 本项目是知识库项目，需要在回答问题或者记录问题的时候，附带通俗易懂的说明；若知识点可配图，需在 doc/pic/ 目录下存放对应 SVG 图片（路径不存在时自动创建），并在正文中以 Markdown 插图语法引用。
2. 本项目需要使用markdown格式记录问题
3. 本项目需要在记录问题的时候，附带问题的标签
4. 本项目需要在记录问题的时候，附带问题的解决方法
5. 在知识库文件后方追加内容时，需要符合上方格式
6. 如果配图需遵循如下要求
    - 总体原则：优先通俗易懂；遇到复杂主题，图示要精辟且详尽，但不堆砌信息
    - 格式与存放：使用 SVG ；文件存放在 doc/pic/ （路径不存在则创建）
    - 信息密度：一图一意，突出核心关系或流程；必要标注到位（标题、关键术语、单位）
    - 视觉可读：颜色对比清晰、字号适中、留白合理；在移动端也能读清
    - 结构优先：少文字说明，多用结构/流程/层级来表达逻辑
    - 命名与引用：文件名采用简短英文主题，如 git-merge-flow.svg ；正文用 Markdown 插图语法引用
    复杂主题处理
    - 先给总览图说明全局关系，再用局部拆解图逐步讲清关键细节
    - 每张图都能独立被理解；层级一致、标注风格统一
